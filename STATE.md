# VoiceDrop — project state (read this first)

Last updated: 2026-06-20

## What it is

VoiceDrop: open-mic-and-record iOS app. You tap record → speak → it uploads the
audio to your own cloud space → a server miner transcribes it and writes one or
more 公众号-style articles in your voice → you read/share them in the app. Plus a
public web preview of any article.

## Architecture & data flow

```
iOS app ──record──> users/<sub>/VoiceDrop-<ts>...m4a  (R2, per-user)
                          │
  GitHub Actions `mine` (hourly cron '17 * * * *' + 加急处理 dispatch)
   └ mine.py: for each unprocessed m4a → Volcano ASR → Claude API → write
       users/<sub>/articles/<stem>.json  (+ .srt)   ← "已成文"
       or users/<sub>/articles/<stem>.empty         ← "无语音" (corrupt/silent/no-speech)
                          │
iOS app `文章` tab ──list+download──> renders articles, share, delete
public web  jianshuo.dev/voicedrop/<token> ──> renders one article set (light theme)
```

Auth: per-user. App holds an anonymous capability token (`anon_…`, iCloud
Keychain) OR a Sign-in-with-Apple session JWT. Server admin token = `FILES_TOKEN`
(sees all `users/*`). Files API scopes every request to `users/<sub>/`.

## R2 layout & marker conventions (the contract everyone shares)

- `users/<sub>/VoiceDrop-<ts>-<dur>-<weekday>-<period>[-<city>-<district>].m4a` — audio (ASCII names).
- `users/<sub>/articles/<stem>.json` — mined article(s), v2 schema `{schema,id,sourceAudio,createdAt,transcript,srt,articles:[{title,body}],status,model}`. **Presence = 已成文.**
- `users/<sub>/articles/<stem>.empty` — `{status:"empty",reason:"corrupt|silent|no-speech"}`. **Presence = 无语音.**
- `users/<sub>/articles/<stem>.srt` — subtitle sidecar.
- `users/<sub>/CLAUDE.md` — the user's name + style (Settings tab). Appended to the mining prompt.
- `shares/<id>` — value is a full article key; backs the short public share link. `id = HMAC(key)[:10]`.
- **"processed" = `.json` OR `.empty` exists.** Audio is NEVER auto-deleted; only the user deletes it in-app.

## Files API (`jianshuo.dev/files/api/<path>`, Cloudflare Pages Function)

`functions/files/api/[[path]].js` — routes (all but `auth/apple` require a token):
- `POST auth/apple` — exchange Apple identity token → session JWT.
- `GET  list` / `GET download/<key>` / `PUT upload/<key>` / `DELETE file/<key>`.
- `GET  share/<articleKey>` → `{url:"https://jianshuo.dev/voicedrop/<id>"}` (signs + stores `shares/<id>`).
- `POST mine` → dispatches the GitHub `mine.yml` workflow (token = `GH_DISPATCH_TOKEN` Pages secret). Powers 加急处理.

`functions/voicedrop/[token].js` — public, unauth. Resolves `shares/<id>` → renders
that one article as a light-theme HTML page. Non-token segment (e.g. `privacy`) →
`context.next()` (static `/voicedrop/` landing + `/voicedrop/privacy/` untouched).

Pages secrets: `FILES_TOKEN`, `SESSION_SECRET`, `GH_DISPATCH_TOKEN`.

## iOS app (`VoiceDropApp/`)

Three tabs (`RootView.swift`, `TabView(selection:)`, reloads on tab activation):
- **录音** `ContentView.swift` — opens **idle** (tap-to-record, not auto-record). State machine: idle → recording → uploading → done. Record button fixed at a constant lower position. Records to a **staging name** `recording-<ts>.m4a` (not `VoiceDrop-*`), promoted to the enriched name only after finalize → fixes the moov-less/0-byte corrupt-upload race.
- **文章** `LibraryView.swift` + `Library.swift` (`LibraryStore`) — lists recordings (badges 已成文 / 无语音 / 待处理). 已成文 rows show the **article title** in place of the place (single line; titles fetched concurrently, cached by articleKey). Swipe-left → red destructive **删除整条录音**. **Easter egg**: long-press the green 已成文 badge → 重新处理这篇文章 (delete article+markers, keep audio, re-mine). `RecordingDetailView.swift` — player + the article shown directly (no 文章/字幕 tabs; the per-article chip switcher shows only when >1 article). Top-right **share** button → system share sheet with the short link.
- **设置** `SettingsView.swift` (`SettingsStore`) — **名字** + **文风** (fixed-height editor, keyboard has a 完成 button). Saves `users/<sub>/CLAUDE.md`. **加急处理** button → `POST /files/api/mine`.

## Server miner (`mining/mine.py`)

- Hourly cron + on-demand dispatch. Idempotent: skips anything with a `.json` or `.empty` marker.
- Per recording: download → ffprobe (corrupt→`.empty corrupt`, <1s→`.empty silent`) → Volcano ASR (empty→`.empty no-speech`) → Claude API (`MINE_MODEL`, default claude-sonnet-4-6) → write `.json`.
- **JSON is forced**: `_articles_from` strips ``` / ```json fences + stray leading `json`, extracts the outermost `{…}`; if unparseable it **raises (retries next cycle)** — never stores raw model text as a body. (Assistant prefill is NOT used — sonnet-4-6 rejects it with 400.)
- Reads the owner's `CLAUDE.md` and appends it after the system prompt.
- ASR (`volc_asr_stream.py`) fast-fails on empty PCM (exit 3) so corrupt files can't hang ~15 min; `transcribe()` has a duration-based timeout.
- Timestamped profiling logs end with `DONE: N mined · M empty · Ts [ASR x% · LLM y% · net z%]` (ASR is the long pole).

## CI / CD & concurrency

- `mine.yml`: `concurrency {group: mine, cancel-in-progress: false}` → **never two mine runs in parallel** (serialized, latest pending wins).
- `build.yml`: `concurrency {group: build, cancel-in-progress: false}` → also serialized, so two builds can't collide on the same `latest_testflight_build_number+1`. Ignores `mining/**` pushes (miner changes don't trigger TestFlight builds).
